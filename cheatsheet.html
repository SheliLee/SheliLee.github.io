Relitive Units
rem
(Root em) Relative to the font-size of the root element (usually ). In most browsers, 1rem equals 16px by default.
em
Relative to the font-size of the element's parent (for typography) or the element itself (for other properties like width).
%
(Percentage) Relative to the parent element's dimensions for that specific property (e.g., width: 50% is half of the parent's width)
vw/vh
(Viewport Width/Height) 1vw is equal to 1% of the browser window's width; 1vh is 1% of its height.
vmin/vmax
Relative to 1% of the viewport’s smaller (vmin) or larger (vmax) dimension.
ch
Relative to the width of the "0" (zero) character in the current font.
fr
distributes the "free space" that remains after any fixed-size items (like pixels or percentages) and grid gaps are accounted for.
Absolute Units
px
(Pixels): The most common unit for screen displays. While traditionally one screen dot, it is formally defined in CSS as 1/96th of an inch.
pt
(Points): Standard for typography; 1pt is 1/72nd of an inch.
cm/mm/in
Centimeters, millimeters, and inches. These are physical measurements mostly used for print stylesheets
Keywords
auto
The browser calculates the size based on default layout rules.
min-content
Shrinks the element to the smallest size possible without overflowing its content.
max-content
Expands the element to its "ideal" size to fit all content on one line if possible.
fit-content
A mix of both that attempts to use the available space but caps at max-content.

Layout Engines Box Modeling Every Element in CSS is a box. The box modeling specifies the rectangle for you to format. This consists of the entire content including the content itself, the inside space (padding), the border around the content, and the outside space (margin). With this you can understand how the element is structured and what parts to modify in formatting for what you need.
<box-sizing:border-box> You can then use <box-sizing: boarder box> to combine these elements together to do the math for you to display the content as actual size when adjusting units for formatting.
Flex Boxes This is a very specific one-dimental layout model for you to arrange elements within a single column or row. it is efficent when wanting to distribute space and align your items within the container itself. While usisng <display: flex> you automatically default all your content to the main horrizontal axis of the page, from there you begin your sytling. There are many properties you can use for further format your content from here.
<flex-direction: > and with this you can change the direction of the content shifting it from one axis to another for formmating on that axis, you do this by setting it to either row or column.
<justify-content: > this give you the ability to move your content along the main axis (horizontal), this is also the default flex direction if you do not change it in flex-direction.
<align-items: > this gives you the ability to move content along the cross access (vertical)
<flex-wrap: > this allows you to formats your items to either wrap when space runs out or hold their position no matter the amount of space available, depending on the fomatting you chooose. No wrap is the deafult.
<align-content: > this only becomes avaiable when you set flex-wrap :wrap. This then opens of the ability to align all content along the cross axis, while holding it all together by default.
<gap: > this gives you the ability to set the gap between the content inside the container.
<flex-grow: > this allows you to select a specific item, series of elements or all elements within your content and expand it to grow to fill the remainder of the container.
<flex-shrink: > this allows you to specific spacific items, or item, to shink at a different rate, or not at all, as compaired to the remaining items within the container as the viewport changes in size.
<flex-basis: > This would be used when your item has a spacific size setting and you want to over ride it.
<flex: > this is the short hand version for grow/shrink/basis combined together. It can be used with the three values in one. When only one value is used, it leaves the rest as default.
<align-self: > this is for an individual item so you can over ride the main align-items and it will be placed different as you specify within the container, along the vertical axis.
<order: > this is so you can change the order the items display within your container. the default of this is 0 and works with positive and negitive numbers as placement. This is not a reecomnded property to use.
Grid This is the two-dimentaioal option that allows you to have more accurate placements of elements usisng a grid system. With this it can utilize both the column and the rows simultaneously you want to arrange your element on. This means you can position elements whatever way you wish where ever you wish, from stacked to spaced out. The system uses numbered row and column lines in a grid pattern to make up square cells and tracks. for this to work you need a container with smaller containers inside, you then set properties to move forward from there. A note to remember: if you are layering do not forget to change your z-index for the layers.
<display: grid> is used to tell the formatting you are setting up a grid to work on.
<grid-template-rows/column: > are used to input how many rowns and columns you with to work with. They define a track for each value. You need to add a value for each row/colum you wish your have in your layout.
<grid-row/column-start/end: > you must use all four properies for each item allow you to position your items within the grid by ususing values
<grid-row/column: > this is the short cut for the four values. This one you need to use two values seperated by a slash. The first one is the start and the second is the ending line.
Alternitivly you can use the smap keyword and define the number of cells you wish your item to span. The problem with this method is you dont define where your item starts so it could push other items to different positions when you span.
<grid-area: > this takes four values: row and colum start, then rown and column end, in that order seprated by a slash. This is the one reccomended to use.
